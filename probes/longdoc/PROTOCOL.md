# Long-Document Regime Protocol

Date locked: 2026-07-04

## Scope

This probe tests whether `md` creates a measurable edge on markdown documents larger
than 10k lines, where native sliced reads can struggle to find a structurally scoped
target.

Task corpus:

- `LD-CHANGELOG-01`: `bench/regimes/longdoc/inputs/mega_changelog.md`
- `LD-RUNBOOK-02`: `bench/regimes/longdoc/inputs/ops_runbook.md`
- `LD-SPEC-03`: `bench/regimes/longdoc/inputs/interop_spec.md`

These are exploratory regime probes. They do not enter `bench/tasks/tasks.json` and
cannot affect v3 headline evidence.

## Fixture Design

Fixtures are generated by `probes/longdoc/generate_fixtures.py` and validated in
`bench/regimes/longdoc/fixture_validation.json`.

Design rules:

- Each input document has at least 10,000 lines.
- Targets are selected by nested structural path plus an ordinal repeated subsection.
- No target uses a unique nearby grep marker. Target heading labels, answer values,
  and checklist/status labels recur many times across each document.
- Expected outputs are compact canonical JSON envelopes so the probe measures
  locating/reading behavior rather than large output copying.

The validation artifact records line counts and repetition counts for each target's
nearby headings/values.

## Arms

- `native`
- `native+md`
- `native+md-no-md`

`native+md-no-md` gets the byte-identical `native+md` prompt with the real `md`
binary stubbed out. It is the prompt/control cell: if it tracks `native+md`, the
effect is not attributable to the tool.

## Models and N

- Strong model: `claude-sonnet-4-6` via `claude-cli`
- Weak model: `claude-haiku-4-5-20251001` via `claude-cli`
- N = 3 per task x mode x model

Hard cap: $40 equivalent subscription/API spend. Abort and report rather than
exceeding it.

## Success Rule

Rows pass only when the emitted JSON envelope matches the expected canonical JSON
after projecting to the pre-registered keys in `tasks.json`.

## Cost and Adoption Metrics

The readout reports:

- pass count and pass^3 per task/model/mode
- average byte-cost proxy: `bytes_prompt + bytes_output + bytes_observation`
- average tool calls and elapsed seconds
- md adoption: rows whose `tool_mix` contains at least one `md ...` verb
- no-md control behavior, especially `native+md` vs `native+md-no-md`

Subscription billing does not expose per-row dollars in the harness, so the byte-cost
proxy is the pre-registered cost axis for this probe.

## Kill Condition

If `native+md` shows no correctness lift and no byte-cost proxy advantage over
`native` on both models, the long-document regime is CLOSED and the falsification
record gets an addendum.

If `native+md` wins on either correctness or byte-cost proxy for either model, the
regime becomes a CANDIDATE family for U5/Plan C. A candidate must still report the
`native+md-no-md` control: if the no-md control matches `native+md`, the evidence is
prompt/strategy, not md-tool attribution.

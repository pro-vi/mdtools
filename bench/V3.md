# mdtools bench v3 protocol

Bench v3 is the replacement measurement protocol for public mdtools benchmark claims.

The pre-v3 headline numbers are retracted. They remain archived in `bench/RESULTS.md`
for provenance, but they used one trial per task/mode, scorer authority coupled to the
tool under test, prompt-coached task descriptions, and gap-selected tasks in headline
tables. Do not cite them as current evidence.

## Publication Rules

- Headline cells require at least five trials per task/mode.
- Published rates include 95% confidence intervals.
- Published comparisons are paired by task.
- Pass@1 mean and pass^k are both reported.
- Cost-vs-success is reported alongside correctness.
- Core and adversarially-mined task splits are rendered separately.
- Scorer divergence blocks headline publication until adjudicated.
- A machine-readable v3 manifest must be frozen before paid headline runs.

## Harness Card Skeleton

The generated v3 canon will fill this section from run bundles and the manifest:

- Runner and runner version
- Model ID
- Tool availability preflight
- Temperature policy
- Trials per task/mode
- Scorer version
- Manifest hash
- Cost basis

Until the generated v3 canon lands, this file defines the protocol only; it does not
contain headline results.
